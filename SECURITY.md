# Security Policy

## Supported versions

| Version | Supported |
|---------|-----------|
| 3.x     | Yes       |
| < 3.0   | No        |

## Reporting a vulnerability

**Do not open a public GitHub issue for security reports.**

Use one of:

1. [GitHub Private Vulnerability Reporting](https://github.com/jayyypatel/rplusview/security/advisories/new) (preferred)
2. Email the maintainer listed in `pyproject.toml` with subject `[SECURITY] rplusview`

Please include:

- Affected version / commit
- Impact (token theft, data exposure, RCE, etc.)
- Reproduction steps or proof-of-concept
- Any known mitigations

## Response SLA

| Step | Target |
|------|--------|
| Acknowledgement | within 72 hours |
| Severity assessment | within 7 days |
| Fix / advisory for high/critical | as soon as practical after assessment |

We will credit reporters who wish to be named, unless you request anonymity.
